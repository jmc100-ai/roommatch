feat(vs-classify): make concurrency + rate_per_min configurable per call

Initial Render run sat at ~0.5 photo/sec with concurrency=8 (35-hour ETA for
Mexico City's 63k photos) — each Gemini call is ~17s end-to-end on Render,
mostly network. Bumping per-call params via API body lets us tune without a
redeploy.

- classify-visual-style.js: accepts opts.concurrency + opts.rate_per_min
  (default 8 / 1000). CLI gets --concurrency / --rate_per_min flags.
- server.js endpoint: forwards concurrency / rate_per_min from body.
- Adaptive log cadence (every 5% or 500 photos, whichever is smaller) so
  long runs don't go silent for minutes between progress messages.
