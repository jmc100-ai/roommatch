v2 Design Rationale

Synthesis of ChatGPT feedback + design exploration v1. The goal: one dominant experience built for "guided discovery → efficient booking."

New: Room Style step New: Search Story Bar New: Command Bar (results only) New: Soft date gating Flow reordered Dates moved to step 4 Options A/B removed

What changed from v1

  • 4 competing patterns → 1 dominant experience
  • Room Style elevated to a full "moment" (not a filter row)
  • Dates moved to step 4 (after emotional investment)
  • Sticky "Search Story" bar replaces breadcrumbs
  • Command Bar now appears only in results (not entry)
  • Soft gating: browse without dates, nudge to add them
  • Options A (accordion) and B (context strip) removed from flow

The new flow order & why

  • City → Neighborhood → Room Style → Dates → Results
  • Emotional before rational: fall in love before logistics
  • Neighborhoods = visual + vibe (kept from v1)
  • Room Style = full-page visual moment (the differentiator)
  • Dates = lightweight, optional, always skippable
  • Results = Command Bar for fast refinement

Search Story Bar (sticky)

  • Always visible once journey starts — solves context loss
  • Shows: Paris › Le Marais › Luxury Suite › Add dates
  • Each segment clickable to jump back and edit
  • Dates segment = call-to-action style until set
  • Replaces wizard-style progress indicators
  • On mobile: bar wraps or abbreviates values, still readable

Room Style cards — design intent

  • Same visual weight as neighborhood cards — a "moment," not a filter
  • 8 styles + "describe yourself" custom option
  • Each card: name, desc, keywords, room-type badge
  • Tapping a card sets the AI query automatically
  • Custom option opens inline text input (no modal)
  • On mobile: 2-column grid, large tap targets, horizontal scroll option

Command Bar (results only)

  • Only appears once user reaches results
  • Optimized for "decision mode" — speed and clarity
  • Shows City / Area / Dates as compact editable chips
  • Dates → inline tray, no navigation away from results
  • Text search always visible for quick re-querying
  • Does NOT appear at entry — removes that confusion from v1

Soft date gating

  • Core principle: let users fall in love first
  • Dates are step 4, not step 3
  • Skip is equally prominent as Confirm
  • In results: price badge shows "Add dates" instead of a number
  • Banner at top of results with single CTA to add dates
  • Prevents "I can't find good dates" from blocking discovery

Mobile strategy

  • Each step = full-width, vertical scroll section (not a modal)
  • Story bar: abbreviated values, stays compact
  • Neighborhood grid: 2-column on small screens
  • Style cards: 2-column grid, large tap targets
  • Dates: stacked single column
  • No popups, no "Next" buttons, no multi-step modals

Open questions for next iteration

  • Should neighborhood be skippable from the story bar directly?
  • Multi-select styles: allow 2+ selections for blended queries?
  • Photo-tap vibes from v1 — where do they live? Below style grid?
  • Story bar on desktop: does it need a "Start over" option?
  • Results page: should style cards reappear as a filter row?
  • Animation between states: cross-fade vs slide-up?
