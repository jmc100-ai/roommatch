Design Options — Trade-offs

Click each tab to interact with the prototype, then come back here for a comparison.

A · Stacked Accordion

  • Always shows all 5 steps — no surprises
  • Locked steps provide a visual "roadmap"
  • Edit any previous step inline — great for iteration
  • Mobile: naturally a single column, works great
  • Best for: users who want to review their choices before searching
  • Taller page — requires more scrolling on mobile
  • Can feel form-like if not carefully styled

B · Context Strip

  • Compact — shows all context in one persistent bar
  • Search box always visible — minimal friction
  • Familiar pattern (similar to Google Flights, Airbnb)
  • Best for: returning users who know what they want
  • The 5 segments can feel cramped on mobile
  • Less visual narrative — neighborhood feels like just another field
  • Hard to surface neighborhood photos in this compact space

C · Command Bar

  • Feels very modern and premium — like a "control center"
  • Sticky/floating — always accessible once results appear
  • Results visible immediately below — great feedback loop
  • Natural for power users who want to tweak filters mid-results
  • Neighborhoods feel like a filter, not a discovery experience
  • Complex to handle gracefully on mobile
  • Room description (free text) and neighborhood compete for bar space

D · Neighborhood First

  • Neighborhood is the hero — gets the visual treatment it deserves
  • Breadcrumb trail keeps progress clear without a wizard
  • Most cinematic feel — matches the premium brand
  • Photo-tap vibes feel natural here alongside neighborhood cards
  • Clean mobile experience — each step is a full screen "moment"
  • More steps before reaching results — slightly higher commitment
  • Dates step currently squeezed into the search tray

Recommendation

  • D + C hybrid — Neighborhood-First page (Option D) for new searches, with a compact sticky Command Bar (Option C style) once results are showing
  • Dates and style can collapse into the search tray below the neighborhood chip
  • On mobile: Option D flows naturally as a series of "moments" — city → area → search
  • On desktop: wider neighborhood grid (3-4 columns), search tray beside the hero text
  • Option A (Accordion) is the simplest to build and the safest for launch if speed matters

Mobile Notes

  • All 4 options are responsive — toggle 📱 in the top bar to preview
  • Option D: neighborhood grid collapses to 2 columns on mobile
  • Option B/C: the horizontal segment bar wraps to 2×2 grid on mobile
  • Dates: native date pickers work best on mobile — avoid custom pickers
  • Vibe presets: horizontal scroll row works perfectly on mobile touch
